## v1.1.0 整改清单（当前优先执行）

### 一、界面与交互优化

#### 1. 左侧导航栏重构
**目标**：用动词+名词表达，让用户一眼知道能做什么

**整改内容**：
- [ ] 顶部高频功能区（优先展示）：
  - 🎨 生成一张海报
  - 🎬 生成一条视频
  - 📝 生成一个公众号文章
  - 📊 生成一份报告
  - 📋 审核一篇文案
- [ ] 后台管理功能折叠或下移（项目管理、文件管理等）
- [ ] 每个图标下方强制显示文字标签
- [ ] 点击直接进入向导模式，不是列表页

**文件**：`src/components/Sidebar.tsx`

---

#### 2. 场景大厅模式
**目标**：/projects 首页从列表改为场景卡片，解决"不知道从哪下手"

**整改内容**：
- [ ] 默认视图改为"场景大厅"（3个大卡片）：
  - 卡片1：做一张海报
  - 卡片2：做一套活动方案
  - 卡片3：审一篇公众号
- [ ] 卡片尺寸：大而清晰，点击区域充足
- [ ] 点击卡片直接进入对应向导
- [ ] 保留列表视图切换入口（右上角）

**文件**：`src/pages/ProjectsPage.tsx`

---

#### 3. 舒适模式开关
**目标**：解决"字太小看不清"的痛点

**整改内容**：
- [ ] 个人设置中增加"舒适模式"开关
- [ ] 开启后效果：
  - 全局字号 +2px（默认15px → 17px）
  - 按钮间距拉大 8px
  - 图标下方强制显示中文文字标签
- [ ] 默认字号从14px提升到15px

**文件**：`src/context/ThemeContext.tsx`、`src/pages/SettingsPage.tsx`

---

### 二、文案降敏（去AI化）

**目标**：回避"人工智能"带来的疏离感和不可靠感

**整改内容**：
- [ ] 全局替换文案：
  - "AI生成" → "辅助写作"
  - "AI分析" → "智能整理"
  - "AI助手" → "写作秘书"
  - "Copilot" → "格式助手"
  - "人工智能" → "智能工具"
- [ ] 减少技术术语使用
- [ ] 避免出现"模型"、"算法"等词汇

**文件**：全局搜索替换

---

### 三、体验层微调

#### 1. 字号优化
**整改内容**：
- [ ] 全局默认字号设为 15px
- [ ] 保留"老年/大屏模式"选项（字号18px+）
- [ ] 确保打印输出清晰

**文件**：`tailwind.config.js`、全局样式

---

#### 2. 图标优化
**整改内容**：

#### 3. 本地下载入口
**目标**：满足"东西要攥在自己手里"的数据主权心理

## v2.0 规划清单（下一版本）

### 战略转型：1+3产品矩阵

#### 1. 角色切换系统
**目标**：登录后根据角色加载不同的功能皮肤和首页视图

**三种角色模式**：

| 角色模式 | 首页视图 | Slogan |
|---------|---------|--------|
| 广告公司模式 | 客户需求看板 | "把客户模糊的想法，变成清晰的工单" |
| 企业品宣模式 | 新媒体日历与资产库 | "老板要的数据，随时拉表；领导要的合规，自动校对" |
| 政府品宣模式 | 政务信息报送台 | "三审三校的智能副驾驶" |

**整改内容**：
- [ ] 用户表增加 role 字段（ad_agency / enterprise / government）
- [ ] 登录后根据角色加载不同的首页组件
- [ ] 角色可在个人设置中切换
- [ ] 不同角色的功能入口和默认配置不同

---

### 三大杀手级场景

#### 2. 甲乙方传译闭环（广告公司专用）
**目标**：终结微信群爬楼，解决语音方阵和潦草手绘理解偏差

**整改内容**：
- [ ] 项目详情页增加"解析客户原话"按钮
- [ ] 支持 .mp3（微信语音）、.jpg（手绘草图）上传
- [ ] AI输出三列表格：
  - 列1：甲方原文/原图关键点（显性需求）
  - 列2：推测意图（隐性需求）
  - 列3：执行动作（自动生成待办）
- [ ] 埋点验证：使用率目标 > 40%

**文件**：`src/components/ClientRequirementParser.tsx`（新建）

---

#### 3. 合规与易读平衡器（政府/国企专用）
**目标**：终结领导审核慢，文稿既合规又易读

**整改内容**：
- [ ] AI文档助手增加"政务新媒体专项检测"开关
- [ ] 双因子评分机制：
  - 红线因子：极限词、落马官员姓氏谐音、日期表述不当
  - 传播因子：下沉市场适读指数，长句占比 > 40% 强制建议拆分
- [ ] "银发友好版"改写功能
- [ ] 导出带"已通过AI初审"水印的Word文档（红头文件格式）

**文件**：`src/components/ComplianceChecker.tsx`（增强）

---

#### 4. 实体交付物生成器（所有用户）
**目标**：终结只看手机不看屏，满足纸质签批习惯

**整改内容**：
- [ ] 项目右上角菜单增加"一键打印汇报夹"
- [ ] 自动聚合：项目名称 + 周期 + 任务进度 + AI阶段性小结
- [ ] 去技术化排版：
  - 黑/白/灰配色
  - 表格框线清晰
  - 禁止深色底纹、低对比度字体
- [ ] 适配老旧激光打印机

**文件**：`src/components/ReportExport.tsx`（已有，增强）

## 关键决策记录

1. **产品定位**：从"智能工作台"→"全能文案库+格式校对秘书"
2. **一句话定位**：输入需求，直接出PPT/海报/策划案
3. **核心策略**：把AI能力"藏"进用户熟悉的工作习惯
4. **命名建议**：《Stratomind 务实版》或《Stratomind 县域专版》
