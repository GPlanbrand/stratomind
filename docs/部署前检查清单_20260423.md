## 2. 后端部署步骤

```bash
# 步骤 1: 进入后端目录
cd vercel-deploy/backend

# 步骤 2: 重新生成 Prisma Client（重要！）
npx prisma generate

# 步骤 3: 同步数据库 Schema
npx prisma db push

# 步骤 4: 构建后端
npm run build

# 步骤 5: 验证构建
ls -la dist/
```

## 3. 数据库迁移

### 新增表
执行 `npx prisma db push` 将创建以下新表：

1. **guest_sessions** - 访客会话
   - 字段: id, token, data, expireAt, createdAt

2. **sms_codes** - 短信验证码
   - 字段: id, phone, code, type, used, expireAt, createdAt

### 修改表
**requirement_sheets** 新增字段:
- projectName
- clientName  
- items (JSON格式)

## 9. 回滚方案

如果部署失败：

```bash
# 后端回滚
cd backend
git checkout HEAD~1  # 回退到上一版本
npx prisma generate
npm run build
railway up

# 前端回滚
cd ..
git checkout HEAD~1
npm run build
vercel --prod
```
