# 灵思AI创意工作台 - 架构调整方案

## 一、当前架构分析

### 1.1 现有路由结构

```
/ (首页)              → HomeLoginPage (登录页)
/home                 → HomeLoginPage (登录页)
/projects             → Layout → ProjectsPage (场景选择页)
/projects/workspace/new → Layout → WorkspacePage (5步流程)
/projects/workspace/:id → Layout → WorkspacePage (5步流程)
/projects/report/:id  → Layout → ProjectReportPage (报告页)
/requirements         → Layout → RequirementsListPage (需求确认单)
/admin                → AdminPage (管理后台)
```

### 1.2 现有布局架构

```
Layout.tsx
├── Sidebar (左侧导航栏)
├── Header (顶部导航栏) - 非workspace页面显示
├── Outlet (页面内容区)
│   ├── ProjectsPage (首页 - 场景选择模式)
│   ├── WorkspacePage (5步流程)
│   ├── ProjectReportPage (报告页)
│   └── ...其他页面
└── AIDialog (全局浮动AI对话框) ⚠️ 问题点
```

### 1.3 当前首页结构 (ProjectsPage.tsx)

```
ProjectsPage
├── 角色欢迎横幅
├── 页面标题 + 下载备份按钮
├── 场景选择区 (4个场景卡片)
│   ├── 品牌全案
│   ├── 活动全案
│   ├── 新品上市
│   └── 内容全案
└── 项目列表 (需要向下滚动查看)
```

### 1.4 AI对话框当前位置

- **位置**：Layout.tsx 第164行，全局渲染
- **问题**：浮动在页面底部，所有页面都显示
- **用户反馈**：希望AI对话框只在报告页显示，用于调整方案

## 六、实现步骤

### 步骤1：调整Layout组件
1. 打开 `vercel-deploy/src/components/Layout.tsx`
2. 删除 `import AIDialog from './AIDialog'` 导入语句
3. 删除 `<AIDialog />` 组件渲染代码（约在164行）
4. 验证其他页面功能正常

### 步骤2：重构ProjectsPage为工作台仪表盘
1. 打开 `vercel-deploy/src/pages/ProjectsPage.tsx`
2. 设计新的UI结构，参考UI规划文档：
   - 创建顶部欢迎区组件
   - 创建仪表盘统计卡片组
   - 创建快捷操作区
   - 保留并优化项目列表
3. 删除scenarios数组和场景选择相关代码
4. 适配新的响应式布局

### 步骤3：在报告页集成AI对话框
1. 打开 `vercel-deploy/src/pages/ProjectReportPage.tsx`
2. 导入AIDialog组件：`import AIDialog from '../components/AIDialog'`
3. 在页面底部添加AI对话框容器区域
4. 配置AIDialog的projectId和项目上下文
5. 调整页面样式，确保AI对话框不影响报告阅读

### 步骤4：创建新组件（可选）
1. 创建 `components/Dashboard/` 目录
2. 实现统计卡片组件
3. 实现环形进度图组件
4. 实现快捷操作组件
5. 在ProjectsPage中引入使用

### 步骤5：样式和交互优化
1. 调整Layout的padding/margin
2. 确保移动端适配
3. 测试页面切换流畅度
4. 验证AI对话框在报告页的功能完整性

## 七、注意事项

### 7.1 数据层面
- 项目数据加载逻辑保持不变
- AI对话的历史记录可考虑按项目隔离存储
- 确保AI对话上下文能访问当前项目数据

### 7.2 性能层面
- 首页重构后注意首屏加载速度
- 统计卡片数据可考虑缓存
- AI对话框采用懒加载

### 7.3 兼容性
- 保持与现有路由的向后兼容
- 保留数据导入/导出功能
- 管理员视角的功能不受影响
