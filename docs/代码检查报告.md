## 三、性能问题

### 3.1 不必要的重渲染风险

| 严重程度 | 文件 | 问题描述 |
|---------|------|---------|
| **P1** | src/components/AIDialog.tsx | 6个独立useEffect，每个都可能触发重渲染 |
| **P1** | src/components/MessageCenter.tsx | loadMessages未使用useCallback缓存依赖 |

#### 详细分析

**AIDialog.tsx 性能问题**:

1. **第99行** - 移动端检测
   ```typescript
   useEffect(() => {
     const checkMobile = () => setIsMobile(window.innerWidth < 768);
     checkMobile();
     window.addEventListener('resize', checkMobile);
     return () => window.removeEventListener('resize', checkMobile);
   }, []);
   ```
   **问题**: resize事件可能频繁触发，建议添加debounce

2. **第107-124行** - 加载历史消息
   ```typescript
   useEffect(() => {
     const stored = localStorage.getItem(STORAGE_KEY);
     if (stored) {
       try {
         const parsed = JSON.parse(stored);
         setMessages(parsed);
       } catch (e) {
         console.error('Failed to load messages:', e);
       }
     } else {
       setMessages([{...默认欢迎消息...}]);
     }
   }, []);
   ```
   **问题**: 组件挂载时同步解析JSON，可能造成阻塞

3. **第463行** - 快捷命令筛选
   ```typescript
   useEffect(() => {
     setFilteredCommands(
       QUICK_COMMANDS.filter(cmd => 
         cmd.trigger.toLowerCase().includes(input.toLowerCase()) ||
         cmd.label.includes(input)
       )
     );
   }, [input]);
   ```
   **问题**: 每次input变化都执行过滤，考虑使用useMemo

**MessageCenter.tsx 性能问题**:

```typescript
const loadMessages = useCallback(async (resetPage = false) => {
  // ...
}, [loading, page, filterType]); // loading不应作为依赖
```
