# 表单简化方案

## 一、当前问题分析

### 1.1 现有流程概览（5步流程）

| 步骤 | 文件名 | 标题 | 核心问题 |
|------|--------|------|----------|
| 第1步 | ClientInfoStep.tsx | 客户背景 | 字段过多（25+字段），分组复杂 |
| 第2步 | RequirementsStep.tsx | 项目需求 | 专业术语多（KPI、触媒习惯等） |
| 第3步 | CompetitorStep.tsx | 竞品分析 | 需要深度调研，门槛高 |
| 第4步 | BriefStep.tsx | 创意简报 | 字段重复汇总，冗余 |
| 第5步 | StrategyStep.tsx | 创意策略 | 偏执行层面，可合并 |

### 1.2 详细字段分析

#### 第1步：客户背景（ClientInfoStep.tsx）

| 字段名 | 类型 | 必填 | 问题分析 |
|--------|------|------|----------|

## 二、简化方案设计

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| **能AI就不填** | 用户只需提供原材料，AI自动生成专业内容 |
| **一句话需求** | 核心信息一句话说清楚 |
| **选大于写** | 用选择代替输入，降低门槛 |
| **大白话优先** | 避免KPI、SWOT等术语 |
| **能说就不打** | 语音输入，降低打字负担 |
| **竞品分析可选** | 不是必做，根据需要选择 |

// 语音识别核心代码
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;

const recognition = new SpeechRecognition();
recognition.continuous = true;      // 连续识别
recognition.interimResults = true;  // 实时显示
recognition.lang = 'zh-CN';         // 中文

recognition.onresult = (event) => {
  const transcript = Array.from(event.results)
    .map(result => result[0].transcript)
    .join('');
  setInputText(transcript);
};

recognition.onerror = (event) => {
  console.error('语音识别错误:', event.error);
  // 降级到手动输入
};

### 10.2 语音输入示例语料

```
✅ 好的输入示例：
- "我们是个火锅店，叫老王火锅，想做开业活动，预算5万"
- "县城有个服装店叫美衣阁，想在抖音上推广一下"
- "我是开培训学校的，想招更多学生，预算不多，2万块左右"

❌ 识别困难的输入：
- "帮我弄一下"（信息不足）
- "看看这个"（无上下文）
- "和上次一样"（需配合上下文）
```
