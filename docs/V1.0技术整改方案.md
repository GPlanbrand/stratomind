## 一、用户状态与权限体系

### 1.1 用户状态定义

| 状态 | 标识字段 | 触发条件 | 数据存储 | 功能权限 |
|-----|---------|---------|---------|---------|
| 访客 | user_status = 0 | 首次访问，未登录 | 仅浏览器本地存储 | 临时工作区，无法云端同步 |
| 体验会员 | user_status = 1 | 输入手机号+验证码 | 云端同步 | 消耗积分使用核心功能 |
| 正式会员 | user_status = 2 | 积分耗尽后完成资料补充 | 云端同步 | 可购买积分，完整功能 |

### 1.2 状态流转逻辑

```
访客 → 触发核心功能 → 弹出登录框 → 输入手机号+验证码 → 体验会员（自动创建账号，赠送600积分）

体验会员 → 积分耗尽 → 弹出积分购买页 → 完善资料 → 正式会员
```

## 二、登录与注册

### 2.1 登录方式
- 主方式：手机号 + 短信验证码
- 备用方式：账号 + 密码（隐藏入口）

### 2.2 注册逻辑
- 不设独立注册页
- 手机号已存在 → 直接登录
- 手机号不存在 → 自动创建账号并登录

### 2.3 验证码发送规则

| 规则项 | 限制值 |
|-------|-------|
| 同一手机号每日发送上限 | 5 次 |
| 发送间隔 | 60 秒 |
| 验证码有效期 | 5 分钟 |

### 2.4 登录态保持
- Token 有效期：7 天
- 支持"信任此设备"选项，延长至 30 天

## 三、积分体系

### 3.1 积分赠送规则

| 触发动作 | 赠送积分 |
|---------|---------|
| 首次手机号登录 | 600 |
| 首次完善资料 | 100 |
| 购买积分包 | 按套餐 |

### 3.2 积分消耗规则

| 消耗动作 | 扣除积分 |
|---------|---------|
| 生成《需求确认单》并截图 | 60 |
| 打包下载项目 | 20 |
| 创建新项目（第4个起） | 10 |

### 3.3 积分包配置

| 积分数量 | 价格（元） | 标签 |
|---------|----------|------|
| 600 | 29 | 体验包 |
| 2000 | 79 | 推荐 |
| 5000 | 169 | 超值包 |

### 3.4 首次购买优惠
- 首次购买额外赠送 200 积分
