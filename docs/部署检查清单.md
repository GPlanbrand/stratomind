## 二、部署步骤

### 2.1 Railway 后端部署

#### 步骤 1：连接 GitHub 仓库
```
1. 登录 Railway 控制台 (railway.app)
2. 点击 "New Project" → "Deploy from GitHub repo"
3. 选择目标仓库
4. 设置项目名称: stratomind-api
```

#### 步骤 2：配置环境变量
```
1. 进入项目 Settings → Variables
2. 添加所有必需的环境变量 (见 1.2)
3. 重要: DATABASE_URL 必须正确
```

#### 步骤 3：部署触发
```
方式 A: 自动部署
- Railway 自动检测 Dockerfile 并构建
- 等待构建完成，获取 URL

方式 B: 手动部署
- railway up --detach
```

#### 步骤 4：验证后端
```bash
# 检查服务状态
curl https://stratomind-production.up.railway.app/api/health

# 预期响应: {"status":"ok","timestamp":"..."}
```

> ⚠️ **记录**: 后端 URL 为 `https://stratomind-production.up.railway.app`

---

### 2.2 Vercel 前端部署

#### 步骤 1：连接 GitHub 仓库
```
1. 登录 Vercel 控制台 (vercel.com)
2. 点击 "Add New..." → "Project"
3. 导入目标仓库
4. Framework Preset: Vite (或 Auto-detect)
```

#### 步骤 2：配置构建选项
```
Build Command: npm run build
Output Directory: dist (或 .next, 根据配置)
Install Command: npm install
```

#### 步骤 3：配置环境变量
```
VITE_API_URL=https://stratomind-production.up.railway.app
```

#### 步骤 4：部署
```
1. 点击 "Deploy"
2. 等待构建完成
3. 获取预览 URL: xxx.vercel.app
```

> ⚠️ **重要**: 部署前确认 `vercel.json` 中的 API 代理地址与 Railway URL 一致

## 四、回滚方案

### 4.1 Vercel 回滚

#### 方式 A：控制台回滚
```
1. Vercel Dashboard → 项目 → Deployments
2. 找到上一个稳定版本
3. 点击 "..." → "Redeploy"
```

#### 方式 B：CLI 回滚
```bash
# 查看部署历史
vercel list

# 回滚到指定版本
vercel rollback [deployment-url]
```

#### 方式 C：Git 回滚
```bash
# 回滚到上一个稳定 commit
git revert HEAD
git push origin main

# 或直接重置
git reset --hard [stable-commit-hash]
git push --force
```

---

### 4.2 Railway 回滚

#### 方式 A：控制台回滚
```
1. Railway Dashboard → 项目
2. 进入 Deployments 标签
3. 点击历史版本 → "Redeploy"
```

#### 方式 B：CLI 回滚
```bash
# 查看部署
railway list

# 回滚到指定版本
railway rollback [deployment-id]
```

#### 方式 C：Docker 镜像回滚
```bash
# Railway 会保留历史镜像
# 在 Railway 控制台选择之前的镜像版本进行部署
```

---

### 4.3 数据库回滚 (如需要)

```bash
# 使用 Prisma 迁移回滚 (谨慎使用)
npx prisma migrate reset

# 或手动恢复备份
psql -h host -U user -d dbname < backup.sql
```

> ⚠️ **警告**: 数据库回滚可能导致数据丢失，务必谨慎操作

## 五、常见问题与解决方案

### 5.1 构建问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `npm install` 超时 | 网络问题 | 使用 `--prefer-offline` 或配置镜像 |
| 构建失败 TypeScript | 类型错误 | `npm run build` 本地排查错误 |
| 找不到模块 | 路径问题 | 检查 `tsconfig.json` 和 `vite.config.ts` |
| Prisma 生成失败 | DATABASE_URL 缺失 | 确保环境变量正确 |

### 5.2 部署问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 404 路由不工作 | `vercel.json` rewrites 错误 | 检查正则表达式和 destination |
| CORS 错误 | 源地址不匹配 | 确认 `Access-Control-Allow-Origin` |
| API 超时 | Railway 冷启动 | 添加健康检查或保持唤醒 |
| 静态资源 404 | 输出目录错误 | 检查 `outputDirectory` 配置 |

### 5.3 运行时问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据库连接失败 | DATABASE_URL 错误 | 检查 Railway 环境变量 |
| 内存溢出 | Node 内存限制 | 添加 `NODE_OPTIONS=--max-old-space-size=512` |
| 端口未监听 | Dockerfile CMD 错误 | 检查 `EXPOSE` 和启动命令 |
| Session 失效 | JWT_SECRET 不一致 | 确保所有实例使用相同密钥 |

## 六、快速检查脚本

### 部署前自检

```bash
#!/bin/bash
# deploy-check.sh

echo "=== 部署前检查 ==="

# 检查环境变量
if [ -z "$DATABASE_URL" ]; then
  echo "❌ DATABASE_URL 未设置"
  exit 1
fi

# 本地构建测试
echo "📦 运行本地构建..."
npm run build

if [ $? -eq 0 ]; then
  echo "✅ 构建成功"
else
  echo "❌ 构建失败，请修复后再部署"
  exit 1
fi

# 检查 vercel.json
if [ -f "vercel.json" ]; then
  echo "✅ vercel.json 存在"
else
  echo "❌ vercel.json 不存在"
  exit 1
fi

echo "=== 检查完成，可以部署 ==="
```
